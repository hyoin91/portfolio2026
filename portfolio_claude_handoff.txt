7. semantic-dark.json 설명
============================================================

semantic-dark는 다크모드용 의미 토큰이다.

중요:
- semantic-light와 같은 이름 구조를 유지한다.
- 값만 다크모드에 맞게 바꾼다.
- 그래야 코드에서 같은 CSS 변수명으로 light/dark 전환 가능.

예:
Light:
--color-background-page: #EDE3FF;
--color-text-primary: #111111;

Dark:
--color-background-page: #17131F;
--color-text-primary: #F8F4FF;

코드에서는 [data-theme="dark"]로 적용할 예정.

9. 토큰 빌드 파이프라인
============================================================

굿소비에서는 index.css처럼 직접 CSS를 썼지만,
이번 포트폴리오는 토큰 JSON을 CSS 변수와 theme 객체로 변환하는 구조를 만들고 있음.

흐름:
Figma Tokens Studio
→ GitHub tokens/*.json
→ scripts/build-tokens.js
→ src/styles/tokens.css
→ src/styles/theme.js
→ React / styled-components

scripts/build-css.js
- tokens/core.json, semantic-light.json, semantic-dark.json, component.json을 읽는다.
- light 토큰은 :root에 생성한다.
- dark 토큰은 [data-theme="dark"]에 생성한다.
- 결과 파일: src/styles/tokens.css

scripts/build-theme.js
- 토큰 구조를 styled-components용 theme 객체로 변환한다.
- 값은 CSS variable 참조 형태로 만든다.
- 결과 파일: src/styles/theme.js

scripts/build-tokens.js
- build-css.js와 build-theme.js를 순서대로 실행하는 최종 실행 파일이다.

실행 명령:
node scripts\build-tokens.js

사용자가 "css 만들어졌어"라고 확인함.
이후 build-css / build-theme / build-tokens 분리 구조를 만들기로 함.

   DnD 기반 폴더 생성·정렬

   localStorage로 검색 기록 저장·삭제

굿소비 강점:
